[Bug]

Corner Pushback (Reverse Sliding) Fix

In fighting games, every successful hit or blocked attack enters hitfreeze first. Once hitfreeze ends, the defender receives pushback (sliding) over several frames. Even if the defender is already touching the corner and cannot move any farther, the pushback must still be processed internally because it keeps the defender in the correct block defense state.

To preserve the proper spacing between both fighters, the game applies reverse sliding to the attacker whenever the defender is trapped in the corner. This ensures that the final distance between both characters matches what would happen if the same attack had connected in the middle of the stage.

The original game handled this incorrectly. Instead of applying reverse sliding independently, it overwrote movement values already stored in memory, including movement generated by special moves. As a result, the attacker was pushed farther than intended, producing incorrect spacing and behavior.

This has now been fixed. Corner reverse sliding now behaves exactly like mid-screen pushback, without overwriting the movement of special attacks. The spacing is now consistent and matches the intended fighting game mechanics.

External Projectile Sliding Fix

This update also fixes an issue involving the bomb.

This bomb is not an attack performed by Player 1 or Player 2. It is an external stage feature created by a background character and is completely independent of the fight itself.

However, the original game incorrectly treated the bomb explosion as if it had been performed by the opposing player. If the explosion was blocked in the corner, the free player would receive reverse sliding as though they had attacked the defender, even though they had no involvement in the hit.

This behavior has been corrected.

Now the bomb is properly recognized as an external projectile that does not belong to either fighter. Blocking the explosion no longer generates reverse sliding on the other player, allowing them to move and attack freely as intended.

Stage Selection Update

The stage selection system has been reverted to its original behavior.
The game will once again use the original stage rotation logic, automatically selecting either Player 1's stage or Player 2's stage, just as in the arcade version. The internal system also prevents the same stage from being played excessively, although it is still possible to play up to two consecutive matches on the same stage before another one is selected.
The previous modification that forced stage randomization every match has been removed. This change restores the ability to play on each character's own stage with its original music.
A new optional stage randomizer has been added. The winning player is the only one who can activate it by holding Input D during the character introduction screen.

Important rules:

The losing player cannot activate or change stage randomization.
A challenger joining the game by inserting a coin and pressing Start cannot randomize the stage.
At the beginning of a new session, when neither player has any wins, Player 1 has priority and may activate stage randomization.
This rule exists because Samurai Shodown does not allow a player to start directly on the Player 2 side. The first player must always begin as Player 1, while Player 2 can only join as a challenger.
This preserves the original arcade stage selection system while giving the current winner the option to choose a random stage when desired

Character Selection Update

Character selection has also been reverted to its original arcade behavior.
Now, once a character is selected, the winning player remains locked to that character, just as in the original game. The only exception involves the Random Character option.
If a player selects a character using Random Select and wins the match, then when a challenger joins, the winning player will be allowed to choose a new character freely. They may select any character manually or use Random Select again.
However, if the player chooses a character manually after that, the original rule applies again: they become locked to that character and cannot switch unless they once again enter the match through Random Select and win.
This change preserves the original arcade experience while providing additional flexibility for players who choose to use Random Select.

Update patch notes

[Nerf]  Shikihadori has been removed from the following characters:
Galford, Hanzo, Nakoruru,Genan,Nicotine, Cham Cham, Earthquake
Reason:
Characters with many special moves while unarmed have lost access to Shikihadori.
The move is now intended to be more exclusive to characters with fewer unarmed special options, 
improving combat balance and character identity.

[System Update] Weapon Timer Saved Between Rounds
New weapon break system update implemented.

Before:
When a weapon was broken, an internal timer controlled when the weapon would return to the player. If the 
round ended before the timer reached zero, the system would reset normally in the next round.

Now:
The weapon recovery timer is saved between rounds.

Example:
If a player still has remaining recovery time when the round ends, the next round will start with that player 
still disarmed until the timer finishes. The timer continues exactly where it stopped in the previous round.

Purpose:
To make weapon breaking more meaningful and preserve the advantage earned by the opponent during combat.

[Versus Mode Update] Stage System Changes
The automatic random stage system in VS mode has been removed.
To restore character identity and allow players to fight on their own stages with their own music themes.

[New Feature] Optional Stage Randomizer
Players with WIN status can now manually randomize stages.

How it works:
After selecting your character, keep holding any input button:
A / B / C / D or any button combination.
Do not release the button after character selection.
Continue holding the input until the VS presentation screen appears, where the announcer introduces both fighters.
The input must remain held continuously for exactly 5 seconds during the transition and VS presentation screen.
If successful, the stage will randomize automatically before the fight begins.
This method prevents timing issues caused by the opponent selecting their character too quickly or too slowly.
This system allows players to choose between:
classic stage rotation or optional random stages.
